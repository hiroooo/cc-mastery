# cc-mastery

**あなたは Claude Code をどこまで使いこなしているか？**

ローカルの `~/.claude` をスキャンして、**RPG風ステータスカード**（5軸レーダー + 総合レベル + 称号 + 推定偏差値）を生成する CLI。依存パッケージゼロ、外部送信ゼロ。

![サンプルカード](docs/sample-card.png)

## 使い方

```bash
npx cc-mastery
```

これだけ。ローカル解析 → 自己完結 HTML レポート生成 → ブラウザで開く。**「カードを PNG 保存」** ボタンで 2400×1350 の X 投稿用画像を書き出せます。

## 5軸

| 軸 | 測るもの |
|---|---|
| **改造** | スキル / スラッシュコマンド / CLAUDE.md / rules / permissions / プラグイン |
| **エージェント化** | agent 定義 / subagent 起動数 / subagent を使うセッション / MCP サーバー |
| **自動化** | hook スクリプト / イベントカバレッジ / hook 実行回数 / スキル発火数 |
| **PDCA** | memory ノート (feedback / reference) / plan ファイル / 設定バックアップ世代 |
| **稼働量** | アクティブ日数 / セッション数 / 出力トークン / プロジェクト数 |

称号は支配的な軸で決まります: **司令塔アーキテクト** / **魔改造の匠** / **全自動の錬金術師** / **改善の求道者** / **不眠の開拓者**…… 全軸 70 超えなら **Claude Code の化身**。レベルに応じてカードの**レア度**（N → UC → R → SR → SSR）も決まりますが、これは★とホロの派手さを変えるだけでスコアには影響しません。

## 採点方式

すべてローカルの実測値から算出 — **乱数なし・外部データなし**。同じ環境なら必ず同じスコアになります。レポート内の「採点方式」パネルにも式を明示しています。

1. **各指標を 0〜100 点に正規化**（飽和曲線）:

   ```
   points = 100 · x / (x + h)
   ```

   `x` = 実測値、`h` = その指標の「50点基準」。`x = h` で 50 点、0 で 0 点、増えるほど 100 に近づくが到達しない。初心者がスキル3個で0に張り付かず、達人がスキル80個でも飽和しない。各 `h` はダッシュボードの軸別内訳に表示。

2. **軸スコア = 指標の加重和**（重みの合計 = 1.0）

3. **総合レベル (1〜99)** — S = 5軸平均:

   ```
   Lv = round( 99 · (S / 100) ^ 0.9 )
   ```

   指数 0.9 で低スコア帯を持ち上げ。Lv99 は事実上到達不能。

4. **推定偏差値** — 想定分布（平均35・標準偏差15）に対して:

   ```
   dev = 50 + 10 · (S − 35) / 15
   ```

   v0.1 は実分布がないため推定値。v0.2 で opt-in の実分布に置換予定。

5. **レア度** — レベルから: SSR (Lv 75–99, ★★★★★) · SR (55–74) · R (35–54) · UC (18–34) · N (1–17)

全指標の `h`・重みの一覧は [`src/scorer.js`](src/scorer.js) の1枚のテーブルに集約。ここを直すとダッシュボードの内訳も自動追従します。

## プライバシー

- ネットワークアクセス一切なし（解析・スコア・描画すべてローカル）
- 共有カードには**集計数値のみ**（スキル名 / プロジェクト名 / パス / モデル名は載らない）
- レポート下半分の詳細ダッシュボードは名前を含むローカル閲覧用（スクショ共有非推奨バナー付き）

## 偏差値は「推定」です

v0.1 の偏差値は想定分布に対する推定値です。v0.2 で opt-in の匿名スコア送信 → 実分布による本物の偏差値・上位N%・リーダーボードを予定しています。

## オプション

`--lang ja|en` / `--output` / `--json` / `--no-open` / `--project` / `--claude-dir` / `--no-cache`

2回目以降はファイル単位キャッシュ（size+mtime）で数秒で完了します。
